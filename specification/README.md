# Specification

This section documents the current specification of the production system — what the system does "as is".

These sections always reflect the system's current state. Changes to that state are introduced through [proposals](../proposals/). See the [contributing guide](../CONTRIBUTING.md) for how a change moves from a proposal into the sections below.

## Sections

- [**Overview**](./overview/): A high-level description of the system, including its mission, the problem it solves, and its scope.

- [**Model**](./model/): The entities in the system's domain model, and how they relate to one another.

- [**Actors**](./actors/): The actors that interact with the system — people, organizations, and external systems — and the hierarchy of permissions between them.

- [**Performance**](./performance/): The system's non-functional requirements: its quality attributes and performance targets.

- [**Features**](./features/): The system's functional requirements, expressed as scenarios in [Gherkin](https://cucumber.io/docs/gherkin/) notation.

- [**Journeys**](./journeys/): User journeys through the system, presented as wireframes and visual designs.
