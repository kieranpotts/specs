# User journeys

This section presents wireframes and visual designs that show how the system's functions are presented to users.

Whereas [features](../features/) specify _what_ the system does, journeys show _how_ those functions come together into a coherent user experience — the screens a user moves through to accomplish a goal.

_Add image assets (PNG, SVG) to this directory and reference them here, grouped by journey. Keep the designs at the level of structure and flow; pixel-perfect visual design is a downstream concern and does not belong in the requirements specification._
